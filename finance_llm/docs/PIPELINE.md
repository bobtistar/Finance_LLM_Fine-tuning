# 파이프라인

## 전체 흐름

```
PDF 수집 → 전처리 → 배치 추출 → LLM 레이블링 → 품질 검토 → Fine-tuning → Multi-agent 편입
```

---

## 1단계: PDF 수집 ✅

- 소스: 네이버 금융 리서치
- 수량: 60개

---

## 2단계: 전처리 ✅

**도구:** `pdf_processor.py` + `vision_processor.py`

| 페이지 유형 | 감지 기준 | 처리 방식 |
|-------------|-----------|-----------|
| 텍스트 페이지 | 일반 텍스트 밀도 | pdfplumber |
| 차트 페이지 | 5자 이하 행 비율 ≥ 50% | Gemini Vision (`gemini-1.5-flash`) |

---

## 3단계: 배치 추출 ✅

- 출력 위치: `output/` 폴더
- 출력 형식: JSON

---

## 4단계: LLM 레이블링 🔄 진행 중

**도구:** `labeler.py`

- 모델: `claude-haiku-4-5`
- 출력: `labeled_dataset.jsonl`
- 분류 체계: 7개 카테고리 (주카테고리 + 보조카테고리 최대 2개)

---

## 5단계: 데이터셋 품질 검토 ⏳ 대기

- 총 레이블링 건수 확인 (목표: 1,000건 이상)
- 카테고리별 분포 확인
- 샘플 50개 수동 검토

---

## 6단계: 소형 모델 Fine-tuning ⏳ 대기

| 항목 | 내용 |
|------|------|
| 모델 | Qwen3.5-9B |
| 방식 | QLoRA |
| 환경 | RunPod |

---

## 7단계: Multi-agent 구조 편입 ⏳ 대기

| 항목 | 내용 |
|------|------|
| 프레임워크 | LangGraph |
| 추론 에이전트 | Claude API |
| 단순 태스크 에이전트 | Fine-tuned Qwen3.5-9B |
